# 🏎️ Corrupto Kart: Fuga del Expediente

**Corrupto Kart: Fuga del Expediente** es un juego de carreras arcade caótico, satírico y retro (8-bits) donde encarnas a diversos presuntos de la corrupción española. Conduce a toda velocidad, arrolla a periodistas "pesados", esquiva charcos de aceite y maletines de dinero, y sobrevive a interrogatorios parlamentarios donde mentir es tu única salida para no confesar y perder la carrera.

---

## 🚀 Características Principales

*   **Jugabilidad Arcade 8-bits**: Física de derrapes fluida, velocidad vertiginosa y partículas explosivas en el Canvas.
*   **Diseño de Sonido Retro**: Audio sintetizado dinámicamente mediante Web Audio API y pista musical triunfal de victoria (`victoria.mp3`).
*   **Muro de la Vergüenza Global (Foro)**: Un foro interactivo conectado en tiempo real a **Supabase** que incluye buscador, paginación veloz y pestañas de filtrado (Recientes, Históricos, Extensos).
*   **Telemetría de Estadísticas**: Registro automático en la base de datos de cada carrera iniciada indicando qué personaje y vehículo han sido seleccionados.
*   **Controles Adaptados**: Soporte completo para teclado, gamepad y joysticks virtuales táctiles para dispositivos móviles.

---

## 🛠️ Tecnologías Utilizadas

*   **Core**: HTML5, Vanilla CSS3 y TypeScript.
*   **Entorno de Desarrollo & Bundler**: Vite.
*   **Base de Datos**: Supabase (PostgreSQL) para persistencia global en tiempo real.
*   **Despliegue**: Vercel.

## ⚙️ Configuración y Ejecución Local

Sigue estos pasos para arrancar el proyecto en tu entorno de desarrollo local:

1.  **Clonar el repositorio o descargar el código**:
    ```bash
    git clone <tu-repositorio>
    cd CorruptoKart
    ```

2.  **Instalar dependencias**:
    ```bash
    npm install
    ```

3.  **Configurar Variables de Entorno**:
    Crea un archivo llamado `.env` en la carpeta raíz del proyecto y añade tus credenciales de Supabase (este archivo está ignorado en `.gitignore` para seguridad):
    ```env
    VITE_SUPABASE_URL=https://xxxxxxxxxxxxxxxxxxxx.supabase.co
    VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
    ```

4.  **Ejecutar en modo Desarrollo**:
    ```bash
    npm run dev
    ```
    Abre la dirección que te proporcione Vite en tu navegador (usualmente `http://localhost:5173`).

5.  **Compilar para Producción**:
    ```bash
    npm run build
    ```
    Esto generará los archivos optimizados listos para desplegar en la carpeta `/dist`.

## 🗄️ Configuración de la Base de Datos (Supabase)

Para tener el foro y las estadísticas de partidas operativos en producción, accede al **SQL Editor** de tu panel de control de Supabase y ejecuta los siguientes comandos SQL:

### 1. Tabla de Comentarios (Muro de la Vergüenza)
```sql
create table comments (
  id bigint generated by default as identity primary key,
  name text not null,
  message text not null,
  timestamp bigint not null,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- Seguridad a Nivel de Fila (RLS)
alter table comments enable row level security;

-- Políticas de Seguridad públicas
create policy "Permitir lectura pública de comentarios"
on comments for select
using (true);

create policy "Permitir inserción pública de comentarios"
on comments for insert
with check (true);
```

### 2. Tabla de Registro de Partidas (Telemetría)
```sql
create table game_plays (
  id bigint generated by default as identity primary key,
  character_id text not null,
  vehicle_id text not null,
  timestamp bigint not null,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- Seguridad a Nivel de Fila (RLS)
alter table game_plays enable row level security;

-- Políticas de Seguridad públicas
create policy "Permitir inserción pública de partidas"
on game_plays for insert
with check (true);

create policy "Permitir lectura pública de partidas"
on game_plays for select
using (true);
```

## 🌐 Guía de Despliegue en Vercel (Paso a Paso)

Para publicar el juego de forma gratuita, sin dominio propio y con total seguridad de credenciales, sigue estos pasos detallados:

1.  **Subir tu código a GitHub**:
    *   Crea un repositorio nuevo en tu cuenta de GitHub (puedes configurarlo como Privado o Público).
    *   Sube tu proyecto local ejecutando los siguientes comandos en tu terminal:
        ```bash
        git init
        git add .
        git commit -m "Initial commit - Corrupto Kart listo"
        git branch -M main
        git remote add origin <URL-de-tu-repositorio-de-github>
        git push -u origin main
        ```
    *   *Nota: Verifica en tu repositorio de GitHub que el archivo `.env` **no** se haya subido. Debería estar completamente ignorado gracias al `.gitignore` configurado.*

2.  **Crear cuenta y conectar en Vercel**:
    *   Regístrate en [vercel.com](https://vercel.com) (se recomienda iniciar sesión con tu cuenta de GitHub).
    *   En el panel de control de Vercel, haz clic en **"Add New"** y luego en **"Project"**.
    *   Importa tu repositorio `CorruptoKart` desde tu cuenta de GitHub conectada.

3.  **Configurar el Proyecto en Vercel**:
    *   **Framework Preset**: Selecciona **Other** o **Vite** (Vercel lo detectará automáticamente).
    *   **Build & Development Settings**: Deja los comandos por defecto (`Build Command: npm run build` y `Output Directory: dist`).
    *   **Environment Variables** (¡CRÍTICO!): Despliega esta sección y añade tus credenciales de Supabase exactamente igual que en el archivo `.env` local:
        *   **Name**: `VITE_SUPABASE_URL` | **Value**: `https://tu-proyecto.supabase.co`
        *   **Name**: `VITE_SUPABASE_ANON_KEY` | **Value**: `tu-anon-key-de-supabase`
        *   *Haz clic en "Add" para cada variable.*

4.  **Desplegar**:
    *   Haz clic en **"Deploy"**.
    *   Vercel instalará las dependencias, compilará la versión de producción en segundos y te otorgará un enlace público gratuito terminado en `.vercel.app` (ej. `corrupto-kart.vercel.app`).

¡Tu web ya estará publicada y completamente operativa de manera segura y gratuita!
